:!toc:

Task-indexed recipes for the day-to-day patterns Graphitron's directives unlock. Browse alphabetically when you know which directive you want; browse by category when you know which problem you're solving.

== Alphabetical

* xref:add-custom-conditions.adoc[Add custom conditions]: attach a developer-supplied predicate to a field, argument, or input field via `@condition`.
* xref:apollo-federation.adoc[Apollo Federation transport]: wire the federation runtime around a Graphitron schema.
* xref:batch-lookups.adoc[Batch lookups]: drive a positional batch fetch with `@lookupKey` and get one result per input position.
* xref:computed-fields.adoc[Computed fields with `@externalField`]: project a derived column from a developer-supplied jOOQ `Field<T>`.
* xref:condition-cascade.adoc[Stacking and overriding conditions]: when explicit `@condition` predicates stack with implicit ones, and what `override:` suppresses.
* xref:connections.adoc[Cursor-paginated connections]: synthesised vs declared connection types, `totalCount` cost, and what the rewrite caps (and doesn't).
* xref:custom-scalars.adoc[Custom scalars]: declare a custom scalar in the SDL and register a `GraphQLScalarType` through the `Graphitron.buildSchema(...)` customizer.
* xref:error-channel.adoc[The errors channel]: route Java exceptions through a service payload's `errors:` field as typed data.
* xref:external-code.adoc[Wire external Java code]: the shared mechanics for `@service`, `@externalField`, `@tableMethod`, `@condition`, `@batchKeyLifter`, `@enum`, and `@record`.
* xref:federation-keys.adoc[Federation key fields]: declare entity keys with `@key`, layer them onto `@node` types, and stub non-resolvable entities owned by another subgraph.
* xref:global-id.adoc[Global object IDs]: Relay Global Object Identification end-to-end, including stable IDs, `keyColumns` migrations, and `Query.node` dispatch.
* xref:handle-services.adoc[Handle services]: bind GraphQL arguments to Java parameters and pick a return shape the framework recognises.
* xref:join-with-references.adoc[Join with references]: pin foreign-key paths between table-bound types so the generator emits a JOIN.
* xref:map-types-to-tables.adoc[Map types to tables]: bind types and fields with `@table` + `@field` across the four directive sites.
* xref:migrating-from-legacy.adoc[Migrating from the legacy generator]: walk a schema authored against `graphitron-codegen-parent` to the rewrite.
* xref:polymorphic-types.adoc[Polymorphic types]: choose between single-table and multi-table layouts for GraphQL interfaces and unions.
* xref:result-types.adoc[Result-type variants]: pick a backing class for a `@record` parent (jOOQ `TableRecord`, jOOQ `Record`, Java record, plain POJO).
* xref:sort-results.adoc[Sort results]: pick a column source, build stable keyset cursors with explicit tie-breakers, and avoid the collation pitfalls.
* xref:split-vs-inline.adoc[When to split queries]: choose between inline correlated subqueries and `@splitQuery`-batched fetches.
* xref:tenant-scoping.adoc[Tenant scoping]: wire `getDslContext` and `getTenantId` on `GraphitronContext` so each request reaches a tenant-specific `DSLContext`.
* xref:test-your-schema.adoc[Test your schema]: wire a consumer-side execution-tier test against a real Postgres.
