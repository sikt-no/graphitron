= Reference
:description: Authoritative descriptions of every directive, plugin parameter, and runtime API entry point.
:!toc:

Authoritative descriptions of every directive, every Maven plugin parameter, every runtime API entry point, and every diagnostic code Graphitron emits.

== Runtime API

* xref:runtime-api.adoc[Runtime API reference]: the `Graphitron.buildSchema(...)` facade (single-arg and federation two-arg overloads), the `GraphitronContext` per-request interface (`getDslContext`, `getContextArgument`, `getTenantId`, `getValidator`), and the `NodeIdEncoder` utility (generic `encode`/`peekTypeId`/`decodeValues` plus per-Node `encode<TypeName>` and `decode<TypeName>` helpers).

== Maven plugin

* xref:mojo-configuration.adoc[Maven plugin configuration]: every parameter on every goal of `graphitron-maven`. Covers the three goals (`generate`, `validate`, `dev`), the shared `<configuration>` parameters (`outputDirectory`, `outputPackage`, `jooqPackage`, `schemaInputs`, `namedReferences`), the `dev`-only watch parameters (`port`, `debounceMs`, `skipInitial`), and the two complex POM bindings (`<schemaInput>`, `<namedReference>`).

== Diagnostics

* xref:diagnostics-glossary.adoc[Diagnostics glossary]: every closed-set diagnostic code the validator emits, organised by category. Covers the three top-level rejection kinds (`author-error`, `invalid-schema`, `deferred`), the nine name-lookup attempt kinds (`column`, `table`, `foreign-key`, `service-method`, `lifter-method`, `enum-constant`, `type-name`, `nodeid-key-column`, `dml-kind`), and the four deferred emit-block reasons.

== Deprecations

* xref:deprecations.adoc[Deprecations]: aggregator for every deprecated directive argument, input field, and whole directive in the rewrite, with migration guidance pointing back at the canonical directive page.
