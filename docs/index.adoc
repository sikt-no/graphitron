= Graphitron
:description: Graphitron is a Java code generator that turns GraphQL schemas + jOOQ database models into resolvers.
:keywords: graphitron, sikt, graphql, jooq, java, code generation
:!toc:

[.hero]
--
[.hero-title]
*A code generator from GraphQL to jOOQ.*

[.hero-lead]
Graphitron generates the wiring between a GraphQL schema and a jOOQ-modelled database, so resolvers, fetchers, and SQL come out of the schema rather than being maintained by hand.
--

[.feature-row, cols="1,1,1", frame=none, grid=none]
|===
|
[.feature]
== Database-first

A well-designed database outlives every application built on top of it. Graphitron works _with_ the database, not around it. Generated code respects constraints; security lives in the database where it belongs.

xref:graphitron-principles.adoc[Read the principles]

|
[.feature]
== Type-safe by construction

Generator code is Java 25; generated output is Java 17. jOOQ-typed table classes mean the SQL you generate has the same compile-time guarantees as the rest of your code.

xref:dependencies.adoc[How jOOQ and GraphQL-Java fit]

|
[.feature]
== Naive about the things that should be

Generated code has no runtime dependency on Graphitron, no business logic, no authorization. When you need custom rules, you drop into hand-written code through explicit extension points.

xref:security.adoc[Security model]
|===
