= Architecture

Contributor-facing documentation for the Graphitron generator: how the pipeline classifies a GraphQL schema, what code it emits, and the design principles that govern both. Start here if you're implementing generators, adding field variants, or fixing bugs in the rewrite pipeline. For end-user documentation (writing schemas, wiring up Maven, running the generator), see xref:../index.adoc[the project home page].

. *xref:code-generation-triggers.adoc[Code Generation Triggers]*: the full classification taxonomy: schema patterns, sealed variants, generated output. This is the reference for what the code does.
. *xref:rewrite-model.adoc[Rewrite Model]*: visual Mermaid diagrams of the sealed type hierarchy: field variants, type variants, and support/composition types.
. *xref:rewrite-design-principles.adoc[Rewrite Design Principles]*: architectural and technical principles governing the rewrite pipeline (generation-thinking, sealed hierarchies, classification boundaries, test tiers, etc.).
. *xref:argument-resolution.adoc[Argument Resolution]*: unified argument classification, `@condition` support, lookup VALUES generation.
. *xref:../roadmap/index.adoc[Rewrite Roadmap]*: remaining generator work. Per-item plan files are authored as markdown in `graphitron-rewrite/roadmap/`; this rendered view (status board, xref:../roadmap/by-theme.adoc[by-theme index], xref:../roadmap/changelog.adoc[changelog]) is regenerated on every build.
